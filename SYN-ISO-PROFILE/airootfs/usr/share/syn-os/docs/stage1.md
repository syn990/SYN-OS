# Stage 1: In-Chroot Configuration

Stage 1 (`syn-stage1.zsh`) runs automatically as the last step of Stage 0, via `arch-chroot "$RootMountLocation" /bin/zsh /usr/lib/syn-os/syn-stage1.zsh`. You do not run it yourself — it depends on state that only exists once you're inside the chroot on a pacstrap'd filesystem.

## Where its input comes from

Stage 1 sources `syn-config.zsh` again, which re-parses the `synos.conf` copy `pacstrapMain` placed on the target (fields like `Hostname`, `Locale`, `Encryption`, `UseLvm`). But disk-specific values resolved at runtime during Stage 0 — `RootFsDev`, `SwapDev`, `LuksUuid` — aren't in `synos.conf` at all. Stage 1 reads them from `/etc/syn-os/install.state`, written by `pacstrapMain` just before the chroot handoff (see [Stage 0](./stage0.md#5-pacstrapmain-syn-pacstrapzsh)). If that file is missing or unreadable, Stage 1 prints an error and exits immediately rather than guessing.

## What it does, in order

1. **Locale / hostname / time / console**: writes `$LocaleGen` to `/etc/locale.gen` and runs `locale-gen`; writes `LANG=$Locale` to `/etc/locale.conf`; writes `$Hostname` to `/etc/hostname`; symlinks `/etc/localtime` to `/usr/share/zoneinfo/$TimeZone` if that path exists, falling back to `Europe/London` otherwise; writes `KEYMAP=`/`FONT=` to `/etc/vconsole.conf`; runs `hwclock --systohc`.

2. **doas/sudo shim**: if `doas` is on `PATH` (it's in `baseCore`, so it always is), writes `/etc/doas.conf` with `permit persist :wheel` (mode 600), replaces `/usr/bin/sudo` with a one-line shim script that execs `doas "$@"` (mode 755), and removes the real `sudo` package with `pacman -Rdd --noconfirm sudo`. `sudo` "works" on the installed system afterward, but it's `doas` underneath.

3. **User account**: requires `UserAccountName` and `UserAccountPassword` to be set (`: "${Var:?...}"`), and refuses to proceed — with an explicit error naming `synos.conf` — if `UserAccountPassword` is still the shipped `CHANGE_ME` placeholder. Creates the account with `useradd -m -G wheel -s "$UserShell" "$UserAccountName"` if it doesn't already exist, then sets its password non-interactively via `chpasswd`. This is what makes the install unattended-capable: no interactive `passwd` prompt to sit and wait for. Immediately after, Stage 1 strips the `UserAccountPassword=` line out of `$SYNOS_CONF` with `sed -i` — the plaintext password only needed to survive long enough to reach this point, and shouldn't stay readable on the installed disk afterward.

4. **Locally-authored native tools**: nothing to build here anymore. `syn-filemanager`, the waybar module backends, `syn-crypter`, and `syn-wifi` are all built once from source at ISO-build time (`BUILD-ARCHISO.zsh`, source under `SYN-SOFTWARE/` — see [Building the ISO](./build/iso-builder.md)) and already copied onto this disk by `pacstrapMain` in Stage 0 (see [Stage 0](./stage0.md#5-pacstrapmain-syn-pacstrapzsh)), before this chroot even started. Stage 1 doesn't touch them at all.

5. **mkinitcpio hooks** (`configure_mkinitcpio`): the base `HOOKS` array is always `base udev autodetect modconf kms keyboard keymap consolefont block`. `encrypt` is appended if `Encryption=yes`, then `lvm2` if `UseLvm=yes`, then `filesystems fsck` unconditionally. The resulting array is written into `/etc/mkinitcpio.conf` via `sed -i "s/^HOOKS=.*/HOOKS=(...)/"`. `mkinitcpio -P` then rebuilds the initramfs with the new hook set — critical for LUKS installs, since the initramfs needs the `encrypt` hook present to actually prompt for a passphrase at boot.

6. **Bootloader**: first builds the kernel command line. If `Encryption=yes`: `cryptdevice=UUID=${LuksUuid}:${LuksLabel} root=${RootFsDev} rw`. Otherwise: resolves the root device's UUID via `blkid -s UUID -o value "$RootFsDev"` and uses `root=UUID=... rw`, falling back to the raw device path if `blkid` can't resolve it. If `SwapDev` is set, a swap UUID is resolved the same way and `resume=UUID=...` is appended for hibernation support. Then, based on `PartitionStrat`:

   - **`uefi-bootctl`**: `bootctl --path=/boot install`; writes `/boot/loader/loader.conf` (`default syn.conf`, `timeout 0`, `editor 0`); writes `/boot/loader/entries/syn.conf` pointing at `/vmlinuz-linux`, with `initrd` lines conditionally prepending `/intel-ucode.img` or `/amd-ucode.img` if present, and an `options` line carrying the constructed root/resume cmdline plus `vconsole.keymap=${KeyMap}` and `$KernelOpts`.
   - **`mbr-syslinux`**: `syslinux-install_update -i -a -m`, then patches the `APPEND` line in `/boot/syslinux/syslinux.cfg` with the same constructed cmdline, if that file exists.
   - **`mbr-grub`**: `grub-install --target=i386-pc --recheck --boot-directory=/boot --modules="part_msdos ext2 biosdisk" "${Disk}"` — targeting the whole disk, not a partition, since BIOS GRUB needs the MBR gap. Hand-writes `/boot/grub/grub.cfg`: `set default=0`, `set timeout=0`, a conditional `background_image /boot/grub/splash.png` block if that file exists (nothing ships one yet, so this degrades to a plain text menu), and a single `menuentry 'SYN-OS'` loading `part_msdos`/`ext2` modules and booting `/vmlinuz-linux` with the same constructed cmdline. GRUB itself never touches encryption here — `cryptdevice=` is resolved by the initramfs's `encrypt` hook at boot time, the same division of labour `uefi-bootctl` uses. This is possible specifically because `mbr-grub`'s `/boot` is its own small unencrypted partition; see [Storage Strategies](./storage-strategies.md#mbr-grub).
   - Any other `PartitionStrat` value is a hard error — `syn-config.zsh` should never let one through, but Stage 1 checks anyway rather than silently doing nothing.

7. **Services**: enables `dhcpcd.service` and `iwd.service` unconditionally (both best-effort — failures are swallowed). If `EnableSsh=yes` in `synos.conf`, also enables `sshd.service` (openssh is always installed via [Package Collection](./packages.md) regardless; this only controls whether it starts automatically). If `systemd-detect-virt` reports `kvm` or `qemu`, also enables `qemu-guest-agent.service` — deliberately conditional, since on real hardware the agent would just idle forever with no virtio-serial channel to talk to, exactly the kind of unnecessary boot noise this project avoids elsewhere (see [Philosophy](./philosophy.md)).

8. **Final banner**: `syn_ui::final_banner` prints completion and prompts for reboot. Control then returns to Stage 0, which had been waiting on the `arch-chroot` call.

## After Stage 1 finishes

Stage 0 returns from `arch-chroot`, copies the full install log onto the installed disk, and the installer script ends. Reboot, remove the install media, and log in as `UserAccountName`. See [Waybar](./waybar.md) and [LabWC](./labwc.md) for the desktop that starts from there.
